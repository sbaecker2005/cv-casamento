🖼️ Guia de Uso das Imagens - Site MonaLisa & João

📁 Estrutura de Pastas Criada

public/images/ ├── hero-bg.jpg (placeholder) ├── placeholder.jpg (placeholder) ├── gallery/ │ ├── casal-1.jpg (seus momentos especiais) │ ├── casal-2.jpg │ └── casal-3.jpg └── venue/ ├── local-1.jpg (fotos do local) ├── local-2.jpg └── local-3.jpg

🎨 Componentes Criados

  • OptimizedImage: Carregamento otimizado com placeholder e fallback
  • ImageGallery: Galeria responsiva com modal de visualização
  • HeroSection: Seção hero com background de imagem

🚀 Como Adicionar Suas Imagens

  1. Otimize suas fotos (use TinyPNG.com ou Squoosh.app)
  2. Renomeie conforme a estrutura:
    hero-bg.jpg - Foto principal do casal (1920x1080) gallery/casal-1.jpg - Primeiro encontro (800x800) gallery/casal-2.jpg - Viagem especial (800x800) gallery/casal-3.jpg - Pedido de casamento (800x800) venue/local-1.jpg - Vista geral do local (800x600) venue/local-2.jpg - Local da cerimônia (800x600) venue/local-3.jpg - Local da recepção (800x600)
  3. Substitua os arquivos placeholder na pasta public/images/
  4. Teste no navegador em http://localhost:5174

📱 Funcionalidades da Galeria

  • ✅ Grid responsivo (1 coluna mobile, 3 desktop)
  • ✅ Modal de visualização em tela cheia
  • ✅ Navegação por setas (← →)
  • ✅ Navegação por teclado
  • ✅ Legendas personalizadas
  • ✅ Lazy loading automático
  • ✅ Fallback para imagens quebradas

🎯 Para Personalizar

No arquivo src/routes/Home.tsx, você pode:

  • Alterar as legendas das fotos no array galleryImages
  • Adicionar/remover imagens
  • Mudar o número de colunas da galeria
  • Trocar a imagem de fundo do hero

⚡ Performance

  • Imagens são carregadas sob demanda (lazy loading)
  • Placeholders aparecem enquanto carrega
  • Fallbacks automáticos para imagens quebradas
  • CSS otimizado para transições suaves
